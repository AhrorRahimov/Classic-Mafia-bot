# 🎭 Mafia Bot

Telegram-бот для игры в мафию с друзьями. Написан на **aiogram 3** с асинхронным
хранилищем на **SQLite + SQLAlchemy 2**, таймерами фаз на **asyncio** и конфигом
из `.env` через **pydantic-settings**.

## ✨ Возможности

- 🎮 Лобби в группе: присоединение кнопкой, старт игры создателем
- 🔀 Автоматическое распределение ролей с балансом под число игроков
- 🌙 Ночные фазы: мафия убивает, детектив проверяет, доктор лечит
- ☀️ Дневное обсуждение и голосование за казнь
- ⏱️ Авто-переходы между фазами по таймеру (с возможностью раннего завершения,
  когда все сделали ход)
- 🏆 Проверка условий победы после каждой фазы
- 📊 Персистентная статистика игрока (победы/поражения/винрейт)
- 🔒 Защита от читерства: мафия не может убить своих, доктор не лечит того же
  игрока дважды подряд, нельзя голосовать за себя и т.д.

## 🚀 Установка и запуск

### Требования
- Python 3.11+ (тестировалось на 3.14)
- Telegram-токен от [@BotFather](https://t.me/BotFather)

### Шаги

1. **Клонируй проект** и перейди в его директорию:
   ```bash
   cd "Mafia bot"
   ```

2. **Установи зависимости** (желательно в виртуальном окружении):
   ```bash
   python -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Создай `.env`** из примера и впиши свой токен:
   ```bash
   cp .env.example .env
   ```
   ```
   BOT_TOKEN=1234567890:AA...твой_токен_от_BotFather
   ```

4. **Запусти бота**:
   ```bash
   python run.py
   ```

5. Добавь бота в группу и отправь `/newgame`.

> ⚠️ Чтобы бот видел сообщения в группе, отключи **Group Privacy** через
> [@BotFather](https://t.me/BotFather) → `/setprivacy` → `Disable`. Иначе
> бот не будет получать текстовые сообщения (команды) из групп.

## 🔄 Как идёт игра

```
/newgame → Лобби (4–10 игроков)
              ↓ /startgame
         Распределение ролей (рассылка в ЛС)
              ↓
         🌙 НОЧЬ (45с) ─── мафия/детектив/доктор выбирают цели в ЛС
              ↓
         ☀️ День: объявление жертвы
              ↓
         💬 Обсуждение (60с)
              ↓
         ⚖️ Голосование (60с) ─── все живые голосуют в ЛС
              ↓
         Казнь или помилование
              ↓
         Проверка победы ─── нет победителя? → снова 🌙 Ночь
              ↓
         🏆 Финал: объявление победителя + раскрытие ролей
```

Фазы могут завершиться раньше таймера, если **все** активные игроки сделали ход.

## 🏗️ Архитектура

Проект построен по принципам **Clean Architecture** с разделением слоёв:

```
app/
├── main.py              # Точка входа: Dispatcher, middleware, polling
├── config.py            # Settings (.env через pydantic-settings)
├── texts.py             # Все пользовательские тексты (HTML)
├── db/
│   ├── models.py        # SQLAlchemy-модели: Game, Player, UserStats
│   ├── session.py       # async engine + sessionmaker + init_db()
│   └── repo.py          # Repository pattern: GameRepo, PlayerRepo, StatsRepo
├── game/
│   ├── enums.py         # Role, GameStatus, GamePhase, Winner
│   ├── constants.py     # Тайминги, лимиты, тексты ролей
│   ├── balance.py       # Распределение ролей по числу игроков
│   └── exceptions.py    # Доменные исключения
├── services/
│   ├── lobby.py         # Лобби + реестр активных сессий
│   ├── session.py       # GameSession: in-memory состояние игры
│   ├── night.py         # Ночные действия и их резолв
│   ├── day.py           # Дневное голосование и его резолв
│   ├── timer.py         # TimerManager (asyncio.Task на игру)
│   └── orchestrator.py  # Оркестратор переходов между фазами
├── handlers/
│   ├── basic.py         # /start /help /cancel /stats
│   ├── lobby.py         # /newgame /join /leave /startgame + inline
│   ├── night.py         # callback'и ночных действий (mafia/detective/doctor)
│   ├── day.py           # callback'и голосования
│   └── router.py        # Сборка всех роутеров
├── keyboards/
│   ├── inline.py        # Сборщики inline-клавиатур
│   └── callbacks.py     # CallbackAction enum + parse_callback()
├── filters/game.py      # IsAlive / IsRole / IsPhase / InLobby / ...
├── middlewares/
│   ├── db.py            # Внедряет AsyncSession в handler data
│   └── services.py      # Внедряет LobbyService + TimerManager
└── states/fsm.py        # FSM-состояния (минимально)
```

### Принципы

- **Слои:** `handlers → services → repo → models`. Хендлеры не трогают SQL.
- **Dependency Injection:** `session`, `games`, `timers` приходят через middleware
  и фильтры, а не через глобальное состояние.
- **Единый источник истины:** активная игра — это `GameSession` в реестре
  `LobbyService`; БД хранит финальные результаты и статистику.
- **Без магических строк:** все действия, статусы, фазы — это `StrEnum`.
- **Типизация:** полная аннотация типов (PEP 484), `from __future__ import annotations`.
- **Безопасность:** `asyncio.Lock` на игру предотвращает race condition при
  одновременных callback'ах; валидация ролей/целей в сервисах.

## ⚙️ Настройка

Параметры в `.env`:

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `BOT_TOKEN` | *(обязательно)* | Токен от @BotFather |
| `DATABASE_URL` | `sqlite+aiosqlite:///mafia.db` | URL БД (async-драйвер) |
| `ADMIN_ID` | `0` | ID админа (0 — отключено) |
| `LOG_LEVEL` | `INFO` | Уровень логирования |

### Тайминги и лимиты

Меняются в `app/game/constants.py`:

```python
MIN_PLAYERS = 4
MAX_PLAYERS = 10
NIGHT_DURATION = 45          # секунд
DAY_DISCUSSION_DURATION = 60
DAY_VOTE_DURATION = 60
```

Таблица баланса ролей — в `app/game/balance.py` (`_BALANCE_TABLE`).

pip install pyflakes

### Расширение

- **Добавить роль:** расширь `Role` в `enums.py`, добавь описание в `constants.py`,
  создай сервис действий (по образцу `services/night.py`) и зарегистрируй его
  в `services/orchestrator.py`.
- **Сменить БД на PostgreSQL:** поменяй `DATABASE_URL` на
  `postgresql+asyncpg://...` и добавь `asyncpg` в `requirements.txt`.
