# План: Деплой бота на Render (Web Service + PostgreSQL)

## Решения (по твоим ответам)
- **Тип сервиса:** Web Service + keep-alive endpoint (чтобы free-тир не засыпал)
- **БД:** PostgreSQL на Render (вместо SQLite)

## Важное замечание
Render free Web Service **засыпает через 15 минут** без входящего HTTP-трафика. Поэтому нам нужен:
1. **HTTP-сервер** на порту `$PORT` (Render этого требует) с endpoint `/healthz`
2. **Внешний cron-пинг** каждые 10-14 минут, чтобы сервис не уснул (например, UptimeRobot, cron-job.org или GitHub Actions)

## Что нужно изменить (минимально инвазивно)

### 1. `requirements.txt` — добавить зависимости
- `asyncpg` — драйвер PostgreSQL (обязательно)
- `aiohttp` — веб-сервер для `/healthz` (уже транзитивно тянулся с aiogram, теперь явно)

### 2. `app/config.py` — адаптация DATABASE_URL
Render инжектит `DATABASE_URL` в формате `postgres://user:pass@host/db` (синхронный). SQLAlchemy с asyncpg требует `postgresql+asyncpg://...`. Добавлю `field_validator`, который автоматически преобразует схему.

### 3. `app/web.py` — новый файл: минимальный HTTP-сервер
- Поднимает `aiohttp.web` на порту из `PORT` (Render) или 8080 (локально)
- Endpoint `GET /` и `GET /healthz` → `{"status": "ok"}`
- Запускается как `asyncio.Task` параллельно с polling'ом

### 4. `app/main.py` — связать всё вместе
- Импортировать и запустить веб-сервер одновременно с polling
- Graceful shutdown обоих

### 5. `render.yaml` — Render Blueprint
Декларативное описание сервиса:
- Тип: `web_service` (free план)
- Runtime: Python 3
- Build: `pip install -r requirements.txt`
- Start: `python run.py`
- Env vars: `BOT_TOKEN`, `DATABASE_URL` (внутренняя от БД Render), `WEBHOOK_ENABLED=false`
- Health check path: `/healthz`

### 6. `.env.example` — обновить (добавить WEB_PORT, прокомментировать DATABASE_URL)
Чтобы локально осталось удобно тестировать.

## Что НЕ меняется
- `app/db/session.py` — миграция уже поддерживает PostgreSQL (там есть ветка `information_schema.columns`)
- `app/db/models.py` — все типы совместимы с PG
- Все handlers/services — работают как есть

## Пошаговый гайд деплоя (после изменений кода)
Я напишу подробную инструкцию в README, включающую:
1. Загрузить проект на GitHub
2. Создать PostgreSQL на Render (бесплатная, 90 дней)
3. Создать Web Service на Render, подключив GitHub-репозиторий
4. Прописать env-переменные (`BOT_TOKEN`, `DATABASE_URL`)
5. Запустить, проверить логи
6. Настроить UptimeRobot для пинга каждые 10 минут
7. Отключить Group Privacy у бота в @BotFather
