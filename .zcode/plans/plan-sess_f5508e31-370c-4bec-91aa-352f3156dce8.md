# План: Telegram-бот для игры в Мафию

## Технологии
- **Python 3.14** + **aiogram 3.x** (актуальная, асинхронная)
- **SQLAlchemy 2.x (async)** + **aiosqlite** — постоянное хранение
- **pydantic-settings** + **python-dotenv** — конфиг из `.env`
- **asyncio.Task** — для таймеров фаз

## Роли
- 🟢 **Житель** — голосует днём
- 🔴 **Мафия** — убивает ночью (1 в игре 4–6 игроков, 2 в игре 7–10)
- 🔵 **Детектив** — проверяет роль игрока ночью (1)
- 🟡 **Доктор** — лечит игрока ночью (1)

## Геймплей (поток)
1. **`/newgame`** в группе → создаётся лобби (inline-кнопка «Присоединиться»).
2. Создатель жмёт **«Начать игру»** (нужно 4–10 игроков).
3. Бот распределяет роли по балансу и шлёт каждому в **ЛС** его роль.
4. **🌙 Ночь (45 сек):** мафия/детектив/доктор выбирают цели кнопками в ЛС.
5. **☀️ День:** бот объявляет убитого (или «никто не умер»), обсуждение 60 сек.
6. **⚖️ Голосование (60 сек):** inline-кнопки; по итогам — казнь или «помилование».
7. После каждой фазы — **проверка победы**:
   - Мафия выигрывает, если `mafia ≥ citizens`.
   - Город выигрывает, если мафия уничтожена.
8. Цикл повторяется, бот объявляет победителя и сохраняет итог в БД.

## Архитектурные принципы (clean code)
- **Разделение слоёв:** handlers → services → repo → models. Хендлеры не трогают SQL напрямую.
- **Dependency Injection** через middleware: `data["session"]`, `data["games"]`.
- **Единый источник истины** по игре — `GameSession` сервис, управляемый `dict[chat_id, GameSession]` в памяти + персистентность в SQLite.
- **Кастомные фильтры** (`IsAlive`, `IsPhase`) вместо копипасты проверок.
- **TypedDict / dataclass / Enum** вместо магических строк.
- **Полная типизация** (PEP 484), `async def` без `asyncio.run` внутри.
- **Без хардкода:** все тайминги и лимиты — в `constants.py` / `config.py`.

## Безопасность и корректность
- Голосовать и действовать могут только живые игроки соответствующей роли.
- Мафия не может убить мафию; детектив видит только роль (мафия/нет).
- Доктор не может лечить того же игрока 2 раза подряд (опционально).
- Защита от двойного голоса и race condition через блокировки `asyncio.Lock` на игру.
- Корректная отмена `asyncio.Task` таймера при `/cancel` или завершении игры.

## Запуск
1. `pip install -r requirements.txt`
2. Скопировать `.env.example` → `.env`, вписать `BOT_TOKEN`.
3. `python run.py`
